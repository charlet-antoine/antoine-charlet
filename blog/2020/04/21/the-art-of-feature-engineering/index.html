Feature Preprocessing¶

Numeric Feature¶

While handling the preprocessing of numeric features, it is important to know whether or not our model depends on the scale of our features. As a rule of thumb, we can categorize models into tree-based models and non-tree based models and keep in mind that while scaling will generally not impact the quality of a tree-based model, it will almost surely impact non-tree based models. Usually, we use preprocessing to scale all features to the same scale, so that their initial impact will be roughly equivalent. But it also can be interesting in some scenario to tune the scaling of some critical features to boost their influences on our predictions. Regularization impact will be proportional to feature scale. Gradient descent can go crazy without proper scaling It is considered as best practice to estimate the proper scaling on the training set and then to apply the same scaling to the test set, as we do not want to use knowledge about the distribution in the test set.

def min_max_scaler(X):
    X_num = X.select_dtypes(include=['float64','int64'])
    return (X_num - np.min(X_num)) / (np.max(X_num) - np.min(X_num))

def std_scaler(X):
    X_num = X.select_dtypes(include=['float64','int64'])
    return (X_num - np.mean(X_num)) / np.std(X_num)

One Hot encoding¶

Mean encoding¶

A "one hot" encoding is the go-to in a fair amount of cases, however when we are dealing with high cardinality category variables there are some drawbacks using this encoding. Indeed, a one-hot encoding on those features will result in highly sparse matrixes and might raise further issues during the training depending on the model we are trying to fit. A mean encoding can be an interesting alternative than a simpler label encoding that would not provide any correlations with the target.

Applying a mean encoding can be worth to try when we want to model complex and non linear feature-target dependencies as its sorting quality will help the classifier to better separate our classes. Tree based models usually don't do well with high cardinality categorical variables due to ther limited depth. With mean encoding, we can compensate for this weakness since the mean encoding allow our tree-based model to reach better loss with shorter tree.

The idea of mean encoding can be extended to generate further features. As always there are no general rules, the best encoding will depend on your situation but below is a list of encoding you may want to try.

\begin{eqnarray} &\text{Weight of Evidence:}& \quad \ln(\frac{Positive}{Negative}) \times 100 \\ &\text{Difference:}& \quad Positive -\, Negative \\ &\text{Sum:}& \quad Positive +\, Negative \\ \end{eqnarray}

def simple_mean_encoding(X, categorical_features, encoding='mean'):
    for col in categorical_features:
        if encoding == 'mean':
            encoding = X.groupby(col).target.mean()
        elif encoding == 'weight_evidence':
            encoding = X.groupby(col).target.apply(
                lambda x: np.log(np.sum(x==1) / np.sum(x==0)) * 100)
        else:
            encoding = X.groupby(col).target.apply(
                lambda x : np.sum(x==1) - np.sum(x==0))
        X[col + '_encoded'] = X[col].map(encoding)
    return X

It is important to proceed with caution when we want to target encode a categorical variable as the risk of introducing a leak in our training set is high and will lead to a terrible overfitting.

  1. Split the training data into a train and a validation set
  2. Estimate our encoding on the train set and apply it on our validation set
  3. Apply a regularised version of our encoding on the train set
  4. Fit a model on our training set and validate
We want to split the data into a train and a validation set before doing anything else. Once done, we will estimate the encoding on the train set and then apply it on the validation set. Need to regularize or overfitting 

from sklearn.model_selection import StratifiedKFold

def cv_mean_encoding(X, categorical_features, n_splits=3, alpha=0):
    skf = StratifiedKFold(shuffle=True, n_splits=n_splits, random_state=123)
    prior = X.target.mean()

    for train_idx, val_idx in skf.split(X, X.target):
        X_train, X_val = X.loc[train_idx], X.loc[val_idx]
        for col in categorical_features:
            mu = X_train.groupby(col).target.mean()
            if alpha:
                nrows = X_train.groupby(col).target.size()
                mu_smoothed = (nrows * mu + alpha * prior) / (nrows + alpha)
                X.loc[val_idx, col + '_encoded'] = X_val[col].map(mu_smoothed)
            else:
                X.loc[val_idx, col + '_encoded'] = X_val[col].map(mu)
    X.fillna(prior, inplace=True)
    return X

def expanding_mean_encoding(X, categorical_features, max_permutation=10):
    n_rows = X.shape[0]
    means = np.zeros((max_permutation, n_rows))
    prior = X.target.mean()
    for col in categorical_features:
        for n_permutation in range(max_permutation):
            permutation = X.reindex(np.random.permutation(X.index))
            cumsum = permutation.groupby(col).target.cumsum() - permutation.target
            cumcount = permutation.groupby(col).target.cumcount()
            means[n_permutation] = (cumsum / cumcount).sort_index()
        X[col + '_encoded'] = np.mean(means, axis=0)
        means = np.zeros((max_permutation, n_rows))
    X[categorical_features].fillna(prior, inplace=True)
    return X
